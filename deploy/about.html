Platform Overview

NSITE MO Viewer is a unified dashboard for the NSITE Management Office that consolidates solution tracking, stakeholder engagement, and action management into a single interface. The platform is fully configurable for deployment by other teams with custom branding, terminology, and color themes.

Getting Started: Common Tasks

Real-world scenarios and how to accomplish them in MO Viewer.

person_add

"I just met someone at DHS who could be a useful contact"

  1. Go to SEP-NSITE
  2. Click Log Engagement
  3. Fill in: Date, Activity Type (Meeting/Conference), their email
  4. Add notes about the conversation and potential follow-up
  5. Their contact will auto-appear in Contacts once engagement is logged
lightbulb If you're at an event, also add them to the event's Guest List in Comms > Events for tracking who attended.
update

"What's new with TEMPO NRT?"

  1. Use Ctrl+K (global search) and type "TEMPO"
  2. Or go to Implementation and find the TEMPO card
  3. Click the card to see: recent updates, stakeholders, milestones
  4. For historical updates, go to Reports > Historical Updates and filter by solution
lightbulb Updates are synced from the weekly Internal and SEP agenda documents.
apartment

"Who are our contacts at NOAA?"

  1. Go to Contacts Directory
  2. Type "NOAA" in the search box
  3. Or go to SEP-NSITE > Agencies view
  4. Expand NOAA in the agency tree to see all contacts and engagement history
lightbulb The heat indicators (hot/warm/cold) show recent engagement activity per agency.
event

"I'm going to AGU next week - how do I prepare?"

  1. Go to Comms-NSITE > Events
  2. Find AGU (or create it if not listed)
  3. Click the event, then go to Guest List tab
  4. Search and add expected attendees from Contacts
  5. Click Generate Prep Report for attendee profiles, talking points, and potential connections
lightbulb Contact profiles include hobbies, interests, and goals - great for conversation starters.
task_alt

"I was assigned an action in Monday's meeting"

  1. Actions are automatically synced from agenda documents
  2. Go to Actions-NSITE to see your assigned items
  3. Filter by your name or by solution
  4. Mark items complete when done - status syncs back to the agenda
lightbulb Use Quick Update to add new action items directly to the agenda document.
flag

"When is OPERA's ORR?"

  1. Go to Schedule for the timeline view
  2. Find OPERA to see ATP, F2I, ORR, and Closeout dates
  3. Or use Ctrl+K and search "OPERA" to go directly to its card
  4. Click the card to see milestone details and memo links
lightbulb Milestone dates come from MO-DB_Solutions and are displayed across multiple views.
article

"We have a great story about HLS - how do I track it?"

  1. Go to Comms-NSITE > Pipeline (Story Pipeline)
  2. Click New Story
  3. Fill in: Title, Solution, Status (Identified/In Progress/Published)
  4. Add target publication and any notes
  5. Track it through the pipeline stages
lightbulb Use Coverage view to see which solutions have comms coverage and identify gaps.
download

"I need a list of contacts for a mailing"

  1. Go to Contacts Directory
  2. Use search/filters to narrow down (agency, solution, role)
  3. Click Export to download as CSV
  4. The export includes: name, email, agency, title, solutions
lightbulb For analytical reports, use Reports page - exports go to Google Sheets with methodology documentation.
slideshow

"I need a presentation about OPERA for an external meeting"

  1. Go to Comms-NSITE > Tools
  2. Click Generate Presentation on the Presentation Builder card
  3. Select OPERA from the solution dropdown
  4. Choose External audience type
  5. Click Generate - presentation opens in new tab
lightbulb Internal presentations include process slides (SEP, onboarding). External presentations focus on impact and value. Slides are filtered automatically based on template metadata.

Security & Access

warning
Contains PII â€” This application displays stakeholder contact information including names, emails, phone numbers, and organizational affiliations.

Hosting Environment

NSITE MO Viewer is a Google Apps Script web application hosted within the OSDI Google Workspace environment. All data remains within Google's infrastructure and is subject to NASA OSDI security policies.

Access Control Method

MO-Viewer uses passphrase + whitelist authentication to support team members with mixed account types (NASA.gov and personal Google accounts):

vpn_key
Layer 1: Team Passphrase Users must know the shared team passphrase to sign in. This is distributed to authorized team members out-of-band (email, Slack, etc.).
checklist
Layer 2: Email Whitelist Users must also be listed in the MO-DB_Access spreadsheet. Both passphrase AND whitelist are required for access.
timer
Session Management Authenticated sessions last 6 hours. After expiry, users must sign in again.
sync
Session Persistence Session tokens are preserved during SPA navigation and browser history changes. Users stay logged in when navigating between pages or using back/forward buttons.

How to Get Access

  1. Contact the NSITE MO team lead to request access
  2. Your email will be added to MO-DB_Access
  3. You'll receive the team passphrase
  4. Visit MO-Viewer, enter your email and passphrase to sign in
security
Authentication Email + Passphrase â€” no Google OAuth required, works with any Google account

Technical Notes

Why passphrase authentication?

Google Apps Script has limitations that prevent standard OAuth-based user identification when deployed as "Execute as: Me":

  • Session.getActiveUser() returns empty for external users
  • Session.getEffectiveUser() returns the script owner, not the visitor
  • Google Identity Services is blocked because GAS serves HTML from sandboxed googleusercontent.com subdomains

The passphrase approach lets users self-identify while the passphrase prevents unauthorized access. Sessions are stored in ScriptCache with unique tokens per user.

Iframe sandbox navigation:

Google Apps Script web apps run in a sandboxed iframe with allow-top-navigation-by-user-activation. This means programmatic redirects (like window.top.location.href) only work when triggered directly by a user gesture (click). After login verification completes asynchronously, we display a "Continue to Dashboard" button instead of auto-redirecting, since the async callback no longer has user activation.

SPA session persistence:

Session tokens are extracted from the URL on page load and stored in Platform.sessionToken. During SPA navigation, the token is included in google.script.history.push() calls so it persists in the browser history. This prevents users from being logged out when navigating between pages or using back/forward buttons.

Presentation Builder: Speaker Notes Metadata

Slide filtering is controlled by metadata in each slide's speaker notes. Add these lines to speaker notes:

Slide: [section], [title]
Audience: internal, external
solution: all
agency: all
Field Values Description
Audience: internal, external, or both Who sees this slide
solution: all or solution IDs Which solutions (comma-separated)
agency: all or agency names Which agencies (comma-separated)

Default: Slides without metadata are included for all audiences/solutions/agencies.

Tip: Changes to speaker notes take effect immediately (no deployment needed).
