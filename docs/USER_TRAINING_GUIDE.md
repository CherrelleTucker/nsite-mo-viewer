## Getting Started

### What is MO-Viewer?

MO-Viewer is the central dashboard for the NSITE Management Office. It provides visibility into:
- **Solutions** across their lifecycle (Preformulation → Operations → Closeout)
- **Stakeholder engagement** through the SEP pipeline
- **Communications** including stories, events, and outreach
- **Team operations** including meetings, availability, and actions

### Accessing MO-Viewer

1. Navigate to the MO-Viewer URL (provided by your team lead)
2. Enter your NASA email address
3. Enter the team passphrase
4. You'll be redirected to the dashboard

### Navigation

The top navigation bar provides access to all major sections:

| Tab | Purpose | Primary Users |
|-----|---------|---------------|
| **Implementation** | Solution lifecycle tracking | Implementation team |
| **SEP** | Stakeholder engagement pipeline | SEP team |
| **Comms** | Stories, events, messaging | Comms team |
| **Contacts** | Stakeholder directory | All teams |
| **Schedule** | Milestone timeline | All teams |
| **Team** | Internal operations | All teams |
| **Reports** | Analytics and exports | All teams |
| **About** | Platform documentation | All teams |

## For Implementation Team

### Your Primary Views

#### Implementation Page
Your home base for tracking solutions across the lifecycle.

**Key Features:**
- **Solution Cards**: Each solution displayed with key status info
- **Phase Filters**: Filter by Preformulation, Formulation, Implementation, Operations, Closeout
- **Cycle Filters**: Filter by assessment cycle (C1-C6)
- **Search**: Find solutions by name or keyword

**Solution Card Details:**
Click any solution card to see:
- Official name and alternate names
- Current lifecycle phase
- Assigned DAAC
- Team contacts (Lead, RA Rep, EA Advocate)
- Decision gate dates (ATP, F2I, ORR, Closeout)
- Key documents (Project Plan, IPA, ICD, etc.)
- Recent updates from meeting notes

#### Schedule Page
Visual timeline of solution milestones.

**How to Use:**
1. View upcoming decision gates across all solutions
2. Filter by cycle or phase
3. Click milestones to see details and related documents

### Common Tasks

**Finding a Solution's Status:**
1. Go to Implementation page
2. Use search or filters to find the solution
3. Click the card to view full details
4. Check "Recent Updates" for latest meeting note excerpts

**Checking Decision Gate Dates:**
1. Go to Schedule page for timeline view, OR
2. Go to Implementation → click solution → scroll to Milestones section

**Finding Key Documents:**
1. Implementation → click solution
2. Scroll to Documents section
3. Click links to open Project Plan, IPA, ICD, etc.

### Data You Generate

Your work in meetings generates data that feeds this dashboard:
- Updates mentioned in **Internal Planning** meetings appear on solution cards
- Decision gate dates you track appear on the Schedule
- Document links you maintain appear in solution details

## For SEP Team

### Your Primary Views

#### SEP Page
The stakeholder engagement pipeline—your command center.

**Key Features:**

**Pipeline View:**
- Solutions organized by their SEP progress
- Visual progression: WS1 → TP4 → WS2 → TP5 → WS3 → TP6 → WS4 → TP7 → WS5 → TP8
- Click solutions to see engagement details

**Solution Detail Modal:**
When you click a solution, you see:
- Current SEP milestone status
- Stakeholder list with contact info
- Engagement history
- Email stakeholders button
- Log engagement button

**Stakeholder Section:**
- View all stakeholders for a solution
- Filter by role (Primary SME, Secondary SME, Survey Submitter)
- Click "Email All" to compose group message
- Log emails as engagements

#### Contacts Page
Your stakeholder directory.

**How to Use:**
1. Search by name, email, agency, or department
2. Filter by solution, role, or survey year
3. Click a contact to see full details and engagement history

### Common Tasks

**Finding Stakeholders for a Solution:**
1. Go to SEP page
2. Click the solution card
3. View the Stakeholders section
4. Use filters to narrow by role

**Emailing Stakeholders:**
1. SEP → click solution → Stakeholders section
2. Click "Email All" to compose to all stakeholders, OR
3. Click individual stakeholder → Email icon

**Logging an Engagement:**
1. After sending an email, click "Log as Engagement"
2. Fill in engagement type, direction, notes
3. Submit to record in engagement history

**Checking Engagement History:**
1. Contacts → search for stakeholder
2. Click their name
3. View "Engagement History" section

**Using Email Templates:**
1. SEP → click solution → Stakeholders → Email
2. In the email composer, use "Template" dropdown
3. Select touchpoint or working session template
4. Template auto-fills with solution context

### Data You Generate

- **Engagement logs** appear in stakeholder histories
- Updates from **SEP Strategy** meetings appear on solution cards
- SEP milestone dates you track appear in the pipeline view

Your hub for stories, events, and outreach tracking.

## For General Team Members

### Your Primary Views

#### Team Page
Your home for internal operations.

**Views (use toggle buttons):**

| View | Purpose |
|------|---------|
| **Overview** | Team profiles + Office availability |
| **Actions** | Track action items from meetings |
| **Kudos** | Give/receive peer recognition |
| **Meetings** | Recurring meeting schedule with links |
| **Documents** | Directing documents (Project Plans, Style Guide, etc.) |
| **Parking Lot** | Ideas and topics for future discussion |

**Overview - Team Profiles:**
- See all team members
- Click a profile to view details
- Filter by team (Implementation, SEP, Comms, etc.)

**Overview - Office Availability:**
- See who's out (vacation, travel, 9/80)
- Office holidays
- Click entries to edit your own
- Add new availability with "Add" button

**Actions:**
- View by Person or by Status (Kanban)
- Filter: Open, All, Done
- Click action to see details and update status
- Add new actions with "Add Action" button

**Kudos:**
- Give recognition to teammates
- Select category (Teamwork, Innovation, Above & Beyond, etc.)
- Optionally post to Slack
- View kudos feed

### Common Tasks

**Adding Your Availability:**
1. Team → Overview
2. Click "Add" next to Office Availability
3. Select type (Vacation, Work Travel, 9/80, etc.)
4. Enter dates and notes
5. Save

**Editing Your Availability:**
1. Team → Overview → Office Availability
2. Click your entry
3. Update dates/notes
4. Save (or Delete)

**Checking Action Items:**
1. Team → Actions
2. Toggle "By Person" to see your assignments
3. Click action to update status

**Giving Kudos:**
1. Team → Kudos
2. Select recipient from dropdown
3. Choose category
4. Write message (280 char max)
5. Check "Post to Slack" if desired
6. Submit

**Finding a Meeting Link:**
1. Team → Meetings
2. Find meeting by day/category
3. Click meeting name for details
4. Use "Join" button for video link

### Quick Tips

- **Refresh**: Click the refresh icon (top right) to reload data
- **Search**: Most pages have search—use it liberally
- **Filters**: Reset filters to see all data if something seems missing

## For Dashboard & Data Admins

### Understanding the Architecture

#### Data Flow
```
Source Documents (Google Docs)
        ↓
   Sync Scripts (Apps Script)
        ↓
   Databases (Google Sheets)
        ↓
   MO-APIs Library
        ↓
   MO-Viewer (Web App)
```

#### Key Databases

| Database | Sheet ID Config Key | Purpose |
|----------|---------------------|---------|
| MO-DB_Solutions | SOLUTIONS_SHEET_ID | Solution master data (48 solutions) |
| MO-DB_Contacts | CONTACTS_SHEET_ID | Stakeholder contacts (4,200+ records) |
| MO-DB_Updates | UPDATES_SHEET_ID | Meeting note updates by year |
| MO-DB_Engagements | ENGAGEMENTS_SHEET_ID | Stakeholder interaction logs |
| MO-DB_Availability | AVAILABILITY_SHEET_ID | Team availability calendar |
| MO-DB_Meetings | MEETINGS_SHEET_ID | Recurring meeting schedule |
| MO-DB_Actions | ACTIONS_SHEET_ID | Action items |
| MO-DB_Config | (Script Properties) | All configuration keys |

#### Configuration (MO-DB_Config)

Central configuration sheet with key-value pairs:
- Sheet IDs for all databases
- Document IDs for source docs
- Integration URLs (Slack webhooks)
- Access control settings

### Admin Tasks

#### Adding a New User

1. Open MO-DB_Access spreadsheet
2. Add row with user's email
3. User can now log in with team passphrase

#### Updating the Team Passphrase

1. Open MO-DB_Config
2. Find key: `SITE_PASSPHRASE`
3. Update value
4. Communicate new passphrase to team

#### Adding a New Solution

1. Open MO-DB_Solutions
2. Add new row with required fields:
   - `core_id` (lowercase, no spaces: "new-solution")
   - `core_official_name`
   - `core_cycle` (C1-C6)
   - `admin_lifecycle_phase`
3. Fill in other fields as available
4. Solution appears in dashboard immediately

#### Updating Solution Data

1. Open MO-DB_Solutions
2. Find solution row by `core_id`
3. Edit fields directly
4. Save (auto-saves in Sheets)
5. Dashboard reflects changes on next load

#### Running Sync Scripts

**Sync Updates from Meeting Notes:**
1. Open MO-DB_Updates spreadsheet
2. Go to Extensions → Apps Script
3. Run `syncWeeklyCurrent` for latest Internal/SEP meeting
4. Check logs for errors

**Sync Historical Data:**
1. Run `syncWeeklyInternalHistorical` for full Internal backfill
2. Run `syncWeeklySEPHistorical` for full SEP backfill
3. These are one-time operations

#### Troubleshooting

**Data not appearing:**
1. Check source sheet for the data
2. Verify column headers match expected names exactly
3. Clear browser cache and refresh
4. Check Apps Script execution logs for errors

**Null responses from API:**
- Response may exceed 5MB limit
- Check function has proper limits/truncation
- Review Apps Script logs

**User can't log in:**
1. Verify email in MO-DB_Access whitelist
2. Confirm passphrase is correct
3. Check MO-DB_Config for ACCESS_SHEET_ID

### Deployment

**To update code:**

1. **Library changes** (library/*.gs):
   - Open MO-APIs Library project in Apps Script
   - Copy updated file content
   - Save
   - Deploy new version if using versioned library

2. **Web app changes** (deploy/*.gs, deploy/*.html):
   - Open NSITE-MO-Viewer project in Apps Script
   - Copy updated file content
   - Save
   - Deploy → Manage deployments → Edit → Version: New version

### Monitoring

**Check for errors:**
- Apps Script → Executions (left sidebar)
- Filter by status: Failed
- Click execution to see error details

**Usage tracking:**
- Apps Script → Dashboard
- View executions over time
- Monitor for unusual patterns

## Appendix: Data Field Reference

### Solution Fields (commonly referenced)

| Field | Description |
|-------|-------------|
| `core_id` | Unique identifier (e.g., "hls", "opera") |
| `core_official_name` | Full solution name |
| `admin_lifecycle_phase` | Current phase |
| `core_cycle` | Assessment cycle (C1-C6) |
| `product_assigned_daac` | Data center |
| `milestone_atp_date` | ATP decision gate date |
| `milestone_f2i_date` | F2I decision gate date |

### Contact Fields

| Field | Description |
|-------|-------------|
| `contact_id` | Unique identifier |
| `first_name`, `last_name` | Name |
| `email` | Email address |
| `department` | Federal department |
| `agency` | Agency/bureau |
| `solution_id` | Linked solution |
| `role` | Role (Primary SME, etc.) |
| `survey_year` | Survey participation year |

### Engagement Fields

| Field | Description |
|-------|-------------|
| `engagement_id` | Unique identifier |
| `contact_id` | Linked contact |
| `solution_id` | Linked solution |
| `activity_type` | Email, Meeting, Call, etc. |
| `direction` | Inbound, Outbound |
| `notes` | Engagement notes |
| `engagement_date` | Date of interaction |
